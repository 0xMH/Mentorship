## Overview
This week focuses on mastering Git & GitHub workflows and getting comfortable with Obsidian for your knowledge management. You'll learn branching, collaboration, and advanced note-taking techniques.

---

## Tasks

### 1. Git Branching & Collaboration
Master the Git workflow used by real development teams.

#### Learning Resources
**Video Learner?**
- [ ] Watch [Git Branching Tutorial](https://www.youtube.com/watch?v=1SXpE08hvGs) (30 mins)
- [ ] Watch [Git & GitHub for Beginners - Complete Course](https://www.youtube.com/watch?v=RGOj5yH7evk) (1 hour)

**Prefer Reading?**
- [ ] Read [Learn Git Branching (Interactive)](https://learngitbranching.js.org/) - Complete the first 4 levels
- [ ] Read [GitHub Flow Guide](https://docs.github.com/en/get-started/quickstart/github-flow)

#### Practice Goals
- [ ] Create a new repository called "git-practice"
- [ ] Make an initial commit with a README file
- [ ] Create a new branch called "feature/add-content"
- [ ] Make changes in that branch and commit them
- [ ] Switch back to main branch
- [ ] Merge your feature branch into main
- [ ] Practice these commands: `git branch`, `git checkout`, `git merge`

**Key Concepts to Understand:**
- What are branches and why use them?
- How to create and switch between branches
- How to merge branches
- What are merge conflicts and how to resolve them

---

### 2. GitHub Collaboration Features
Learn how teams work together on GitHub.

#### Practice Goals
- [ ] Fork a public repository (try forking a simple project)
- [ ] Clone your forked repository to your local machine
- [ ] Make a change and push it to your fork
- [ ] Create a Pull Request (PR) from your fork
- [ ] Explore GitHub Issues on any public repository
- [ ] Star repositories you find interesting

#### Learning Resources
**Video Learner?**
- [ ] Watch [Pull Requests Explained](https://www.youtube.com/watch?v=8lGpZkjnkt4) (20 mins)

**Prefer Reading?**
- [ ] Read [About Pull Requests](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests)
- [ ] Read [Creating an Issue](https://docs.github.com/en/issues/tracking-your-work-with-issues/creating-an-issue)

**Bonus Challenge:**
- [ ] Find an open-source project with "good first issue" label and try to contribute
- [ ] Create a GitHub profile README (it's a special repository!)
